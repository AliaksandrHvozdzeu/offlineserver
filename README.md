# Offline Connect Server

Сайт и **офлайн-чат** для локальной Wi‑Fi сети модема (например **Huawei E5573**). Все, кто подключён к Wi‑Fi модема, видят один и тот же сайт и общий чат — без интернета.

## Структура проекта

- **`server/`** — Node.js сервер: раздаёт собранный React-сайт и WebSocket для чата.
- **`client/`** — React (Vite) приложение: главная страница и чат.

На самом модеме E5573 сервер не запускается (закрытая прошивка). Сервер нужно запускать на устройстве в той же Wi‑Fi сети (ноутбук, Raspberry Pi и т.п.).

## Требования

- Node.js 14+
- Устройство в сети Wi‑Fi модема для запуска сервера

## Установка

```bash
# Из корня проекта
npm run install:all
```

Или по отдельности:

```bash
npm install
cd server && npm install
cd ../client && npm install
```

## Сборка и запуск

1. Собрать фронтенд:

```bash
npm run build
```

2. Запустить сервер (раздаёт сайт с папки `client/dist` и WebSocket для чата):

```bash
npm run start
```

В консоли появятся адреса вида `http://192.168.8.xxx:3000`. Откройте любой из них в браузере на устройствах в той же Wi‑Fi сети.

Порт можно задать через переменную окружения:

```bash
PORT=8080 npm run start
```

## Разработка

- **Клиент (React):** `npm run dev:client` — Vite на порту 5173, горячая перезагрузка. Чат в режиме разработки подключается к `ws://localhost:3000`, поэтому сервер тоже должен быть запущен.
- **Сервер:** из папки `server` — `node server.js` (порт 3000). После изменений в React пересоберите: `npm run build`, затем перезапустите сервер.

## Как это работает

1. Модем раздаёт Wi‑Fi (например «Huawei-…»).
2. На ноутбуке или Raspberry Pi в этой сети запускается этот сервер (HTTP + WebSocket).
3. Пользователи подключаются к Wi‑Fi и открывают в браузере `http://<IP-сервера>:3000`.
4. Все видят один сайт и один общий чат в реальном времени (WebSocket).
